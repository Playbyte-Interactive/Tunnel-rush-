# Game Build Checklist

Use this before sending any game link.

- The first screen is playable within a few seconds.
- The game has a distinct theme from the reference link.
- Desktop controls work.
- Mobile touch controls work.
- The game has a score, distance, progress, or equivalent success metric.
- The player can lose or finish a run.
- The player can restart immediately after game over.
- HUD does not cover the core playfield.
- Layout works on a phone-sized viewport.
- Build passes with `npm run build`.
- A quick browser playtest confirms that the canvas is not blank and input changes gameplay.

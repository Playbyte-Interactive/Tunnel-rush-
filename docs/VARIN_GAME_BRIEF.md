# Varin Game Brief

Use this as the shared checklist for each game in this batch.

- Build small, polished 2D games for a TikTok-like mobile game feed.
- Default stack: React + TypeScript, with fast load times.
- Avoid Unity web builds for now because instant play matters.
- Each game must be playable end-to-end with a clear gameplay loop.
- Include a start screen, scoring or progress tracking where relevant, game-over and restart flow, responsive controls, and no major bugs.
- Make games mobile-friendly and understandable within a few seconds.
- Prefer casual or hypercasual mechanics: simple, addictive, quick replay.
- Do not copy reference game assets or exact themes; use references for mechanics only.
- Open-source or generated assets are allowed only when licensing is safe.
- Final quality is judged by playability, fun, visual polish, mobile support, core-flow stability, closeness to the brief, and reasonable code quality.
- Expected pace: roughly one game every 1-2 working days, with 1-2 revision rounds per game.
